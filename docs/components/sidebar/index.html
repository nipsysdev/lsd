Sidebar

Collapsible navigation sidebar with multiple variants and responsive design

About Sidebar

Sidebars are navigation components that provide access to different sections of your application. They support multiple visual variants, collapsible modes, and responsive behavior for optimal user experience across devices.

Installation

Import the Sidebar components from LSD:
import {
  Sidebar,
  SidebarProvider,
  SidebarContent,
  SidebarHeader,
  SidebarFooter,
  SidebarInset,
  SidebarTrigger,
  SidebarRail,
  SidebarSeparator,
  SidebarInput,
  SidebarGroup,
  SidebarGroupLabel,
  SidebarGroupAction,
  SidebarGroupContent,
  SidebarMenu,
  SidebarMenuItem,
  SidebarMenuButton,
  SidebarMenuAction,
  SidebarMenuBadge,
  SidebarMenuSub,
  SidebarMenuSubItem,
  SidebarMenuSubButton,
  SidebarMenuSkeleton,
} from '@nipsys/shadcn-lsd'

export default function MyComponent() {
  return (
    <SidebarProvider>
      <Sidebar>
        {/* Sidebar content */}
      </Sidebar>
    </SidebarProvider>
  )
}

API Reference

All available props for the key Sidebar components.
variant
The visual style of the sidebar
Type: 'sidebar' | 'floating' | 'inset'
collapsible
The collapsible behavior of the sidebar
Type: 'offcanvas' | 'icon' | 'none'
side
The side of the screen where sidebar appears
Type: 'left' | 'right'
defaultOpen
Initial open state of the sidebar
Type: booleanControls whether sidebar is open on initial render
open
Controlled open state of the sidebar
Type: booleanUse with onOpenChange for controlled component
onOpenChange
Callback when sidebar open state changes
Type: (open: boolean) => voidCalled when sidebar is opened or closed
isActive
Active state for menu buttons
Type: booleanHighlights the menu item as currently active
tooltip
Tooltip text for menu buttons
Type: stringShows tooltip on hover, useful for collapsed sidebar
showIcon
Show icon in skeleton loading state
Type: booleanDisplays icon placeholder in skeleton
variant (MenuButton)
Visual variant for menu buttons
Type: 'default' | 'outline'
size (MenuButton)
Size variant for menu buttons
Type: 'default' | 'sm' | 'lg'

Accessibility

The Sidebar component follows accessibility best practices.
Keyboard Navigation
Full keyboard support for navigation
• Tab - Navigate to the sidebar• Shift + Tab - Navigate to previous focusable element• Arrow Keys - Navigate between menu items• Enter / Space - Activate menu item• Escape - Close sidebar (in offcanvas mode)• Ctrl/Cmd + B - Toggle sidebar (keyboard shortcut)
ARIA Attributes
Proper ARIA attributes for screen readers
• Sidebar uses role="navigation" for semantic meaning• Menu items use role="menuitem" for proper navigation• Active states are indicated with aria-current="page"• Expanded/collapsed states use aria-expanded"• Tooltips use aria-label" for icon-only buttons
Focus States
Visible focus indicators for keyboard users
Sidebar components have visible focus states that follow the LSD design system's focus indicators, ensuring keyboard users can always see which element has focus. Focus management is handled automatically when opening/closing the sidebar.
